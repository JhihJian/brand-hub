# 节物 Brand Hub Web 设计文档

## 概述

为 Brand Hub 项目添加前端页面，提供用户登录、注册、个人信息管理和后台管理功能。

## 技术选型

- **前端**：纯 HTML + CSS + JS
- **部署**：Fastify 静态托管 `public/` 目录
- **风格**：遵循节物设计规范（出版物品质，灰度为主，无 emoji/阴影/渐变）

## 文件结构

```
public/
├── index.html              # 入口页面（重定向）
├── login.html              # 登录页
├── register.html           # 注册页
├── profile.html            # 用户信息页
├── admin/
│   ├── index.html          # 管理后台首页（用户列表）
│   ├── invitations.html    # 邀请码管理
│   └── user-detail.html    # 用户详情页
├── css/
│   └── style.css           # 统一样式
└── js/
    ├── api.js              # API 调用封装
    ├── auth.js             # 认证状态管理
    └── pages/              # 各页面逻辑
```

## 路由设计

Fastify 通过 `@fastify/static` 托管 `public/` 目录：

| 页面 | 路径 | 认证要求 |
|------|------|----------|
| 入口 | `/` | 重定向至登录或个人信息页 |
| 登录页 | `/login.html` | 无 |
| 注册页 | `/register.html` | 无 |
| 用户信息页 | `/profile.html` | 需登录 |
| 用户管理 | `/admin/index.html` | 需 admin 角色 |
| 邀请码管理 | `/admin/invitations.html` | 需 admin 角色 |
| 用户详情 | `/admin/user-detail.html?sub=xxx` | 需 admin 角色 |

## 认证机制

### Token 存储

- `access_token`：存储在内存中（不持久化）
- `refresh_token`：存储在 `localStorage`
- 用户信息：存储在 `sessionStorage`

### 认证守卫

1. 访问 `/profile.html` 和 `/admin/*` 时检查 `refresh_token`
2. 若无 token，重定向到登录页
3. 若 token 有效但 access_token 过期，自动刷新
4. 管理员页面额外检查 `roles` 包含 `admin`

## API 对接

### 认证相关

| 操作 | 方法 | 端点 |
|------|------|------|
| 发送验证码 | POST | `/auth/sms/send` |
| 注册 | POST | `/auth/register` |
| 登录 | POST | `/auth/login` |
| 刷新 Token | POST | `/auth/refresh` |
| 登出 | POST | `/auth/logout` |

### 用户相关

| 操作 | 方法 | 端点 |
|------|------|------|
| 获取个人信息 | GET | `/users/me` |
| 更新昵称 | PATCH | `/users/me` |

### 管理员相关

| 操作 | 方法 | 端点 |
|------|------|------|
| 批量生成邀请码 | POST | `/admin/invitations/batch` |
| 获取邀请码列表 | GET | `/admin/invitations` |
| 获取用户列表 | GET | `/admin/users` |
| 获取用户详情 | GET | `/admin/users/:sub` |
| 更新用户状态 | PATCH | `/admin/users/:sub/status` |
| 授予会员 | POST | `/admin/users/:sub/membership` |

## 设计规范

遵循节物品牌设计规范：

- **色彩**：灰度阶梯为主，语义色极度克制
- **字体**：PingFang SC, Noto Sans SC, Helvetica Neue
- **间距**：基于 4px 倍数
- **禁止**：无 emoji、无阴影、无渐变、无动画

## 实现步骤

1. 创建 `public/` 目录结构
2. 实现通用样式 `css/style.css`
3. 实现 API 封装 `js/api.js`
4. 实现认证模块 `js/auth.js`
5. 实现各页面 HTML 和 JS
6. 配置 Fastify 静态服务
7. 测试各页面功能
